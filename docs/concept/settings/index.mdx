# Configuration Server

> The central brain of layline.io that manages global settings, user access, security credentials, and cluster coordination.

## Overview

The Configuration Server is the administrative control plane for your layline.io deployment. While Projects and Workflows define how data flows through your systems, the Configuration Server governs who can access what, how credentials are secured, and how multiple processing clusters are coordinated.

Think of it as the operating system layer: it sits beneath your Workflow configurations, providing identity management, secrets storage, and infrastructure connectivity that all Projects share.

## When to Configure the Configuration Server

**Configure these settings when:**

- You're setting up layline.io for the first time and need to define administrative users
- You're adding team members who need access to create or manage Projects
- You're connecting to external systems that require TLS certificates or SSH keys
- You're deploying multiple Reactive Engine Clusters and need to register them
- You're configuring SSO, LDAP, or other authentication integrations

**The Configuration Server is NOT where you:**

- Design Workflows (that's in [Projects](../projects-workflows/))
- Configure data processing logic (that's in [Assets](../../assets/))
- Monitor running engines (that's in [Operations](../../operations/))

## Key Concepts

| Concept | Description |
|---------|-------------|
| **User Storage vs. Cluster User Storage** | The Configuration Server has its own user store separate from each Reactive Engine Cluster. A user with admin rights on the Configuration Server does NOT automatically have rights on processing clusters. |
| **Security Storage Scope** | Credentials stored here are available to the Configuration Server itself (for Git operations, external API calls, etc.), not to running Workflows. Workflow credentials are configured per-Cluster. |
| **Cluster Registration** | Before a Reactive Engine Cluster can be managed, it must be registered in Cluster Storage. This establishes the trust relationship between control plane and data plane. |
