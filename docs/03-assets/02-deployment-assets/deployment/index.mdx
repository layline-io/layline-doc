# Deployment Assets

Deployment Assets define how and where workflows are deployed to Reactive Engines, and how resources are managed at runtime.

## Asset types

| Asset | Purpose |
|-------|---------|
| [Engine Configuration](asset-deployment-engine) | Defines which workflows and assets are deployed to a specific engine |
| [Scheduler Settings](asset-deployment-scheduler) | Per-workflow resource limits (CPU, memory, concurrent instances) |
| [Tag Settings](asset-deployment-tag) | Tags for filtering and organising engines and deployments |
| [ReactiveCluster](asset-deployment-cluster) | Cluster configuration for coordinating multiple engine instances |
| [Deployment Composition](asset-deployment-composition) | Groups all deployment configuration into a single deployable unit |

## Related Resources

Secrets and environments are configured as [Resource Assets](../resources/asset-resource-secret) and [Environment Resource Assets](../resources/asset-resource-environment) respectively — referenced here in deployment configurations rather than duplicated.
